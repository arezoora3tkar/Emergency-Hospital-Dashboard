# داشبورد اورژانس بیمارستان (Emergency Hospital Dashboard)

این داشبورد Power BI برای تحلیل داده‌های اورژانس بیمارستان ساخته شده. تمرکز روی زمان انتظار بیمار، سطح رضایت، و توزیع دپارتمان/نژاد (داده‌های ۲۰۲۴). با KPIها مثل ۷۳۳ بیمار، متوسط زمان انتظار ۳۵.۳ دقیقه، و رضایت ۴.۹۸/۵، می‌تونی روندها رو فیلتر و تجسم کنی. مناسب برای مدیران بیمارستان یا تحلیل‌گران بهداشت.

## ویژگی‌های داشبورد:
- **فیلترهای دینامیک:** بر اساس ماه، دپارتمان (Neurology, General و غیره)، نژاد، و زمان.
- **چارت‌ها و KPIها:** 
  - جدول اطلاعات بیماران و توزیع نژاد/دپارتمان (Bar/Pie).
  - کلیات: KPIهای اصلی (تعداد بیمار، متوسط زمان انتظار، رضایت).
  - نمایش ماهانه: خط زمان (Line Chart) و تقویم (Calendar View).
- **داده‌ها:** بر اساس داده‌های واقعی اورژانس (۷۳۳ ردیف، شامل Patient ID, Wait Time, Satisfaction, Department و غیره).

## فایل‌ها:
- **Hospital ER_Data.csv**: فایل داده‌های خام (۷۳۳ ردیف CSV).
- **Hospital.pbix**: فایل اصلی داشبورد Power BI (شامل تمام ویژوال‌ها و فیلترها).
- **README.md**: توضیحات پروژه.
- **images/**: پوشه اسکرین‌شات‌های داشبورد (PNG/JPG برای پیش‌نمایش).

## چطور استفاده کنی:
1. فایل **Hospital.pbix** رو دانلود و باز کن (Power BI Desktop 2024+).
2. فیلترها (بالای داشبورد) رو انتخاب کن مثل Neurology یا January – ویژوال‌ها خودکار آپدیت می‌شن.
3. **Refresh** بزن تا داده‌ها از CSV لود بشه.
4. برای ویرایش: داده‌های CSV رو تغییر بده و Publish کن.
5. اگر Power BI نداری، از Power BI Service آنلاین استفاده کن (آپلود فایل).

## یافته‌ها و پیشنهادها:
- بیشترین زمان انتظار: دپارتمان Neurology (۴۵ دقیقه).
- رضایت بالا: دپارتمان General (۵/۵).
- پیشنهاد: هشدار خودکار برای زمان انتظار >۳۰ دقیقه، و بهبود توزیع منابع بر اساس نژاد/دپارتمان.

## مشکلات رایج و حل:
- **ویژوال‌ها آپدیت نمی‌شن؟** Refresh data رو بزن یا فیلترها رو پاک کن.
- **Map یا جدول خطا می‌ده؟** داده‌های نژاد/دپارتمان رو چک کن (انگلیسی باشن).
